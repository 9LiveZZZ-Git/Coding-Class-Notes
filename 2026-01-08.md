# Concepts







- [Deterministic](https://en.wikipedia.org/wiki/Deterministic_system)

  - a deterministic system is a system in which no randomness is involved in the development of future states of the system. A deterministic model will thus always produce the same output from a given starting condition or initial state.

  - also "relating to the philosophical doctrine that all events, including human action, are ultimately determined by causes regarded as external to the will."

- [Pseudorandom](https://en.wikipedia.org/wiki/Pseudorandomness) (looks random to a human, but is deterministic)

  - appears to be statistically random, despite having been produced by a completely deterministic and repeatable process

    ```c++
    float rnd() {
      static uint32_t state = 0U;
      state += 12345U;
      state *= 1103515245U; // overflow
      return state / 4294967296.0f;
    }
    ```

  - 0.827574f, 0.652404f, 0.837477f, 0.0533814f, 0.758284f

  - or.. [xorshift](https://en.wikipedia.org/wiki/Xorshift)

- [Recursion](https://en.wikipedia.org/wiki/Recursion) (using a thing within its own definition w/o tautology)

  ```js
  function factorial(n) {
    if (n < 2) {
      return 1;
    }
    else {
      return n * factorial(n - 1);
    }
  }
  
  function fibonacci(n) {
    if (n < 2) {
      return 1;
    }
    else {
      return fibonacci(n - 1) + fibonacci(n - 2);
    }
  }
  
  function gcd(a, b) {
    if (b == 0) {
      return a;
    }
    else {
      return gcd(b, a % b)
    }
  }
  ```

  Recursion is considered more "elegant" or "mathematically beautiful" than alternatives because of its succinctness and because it seems [self-similar](https://en.wikipedia.org/wiki/Self-similarity), like a fractal. Crucially, the recursive call must make the process "smaller" in some way.

- [Enumeration](https://en.wikipedia.org/wiki/Enumeration) (assigning numbers to a set of things)

  - In the fractal (recursive) definition of the circle drawing scheme, each drawing has a number. In general, every program has a number. This number can be all the bits of the program source code arranged in order, taken as a large number. We can ennumerate all programs this way.
  - In Wolfram's minimalist cellular automata example (_A New Kind of Science_ § 3.2), he enumerates 256 programs based on the essential ruleset of his program. This is your reading!

- [Compressibility](https://en.wikipedia.org/wiki/Data_compression) (reducing data size without losing information)

  One way to find a shorter program for an object is to compress the object. We saw this with the PNG of a circle. We found a program of 118 bytes, a PNG of about 4.7k bytes and a full set of pixels with 40k bytes. The compressed image did not represent the smallest source program, but it was smaller than the raw bitmap image. To what degree can an object be compressed? Is a highly compressable object a low-complexity object?

  Consider `noise.png` a compressed version of an image of colorful noise (as opposed to a red circle, simple in color and shape). We have 35k bytes compressed versus 40k bytes uncompressed, a much higher Kolmogorov complexity.

  Now, consider the [source code](https://editor.p5js.org/kybr/sketches/f-55WoLi0)\* of 587 bytes. It would be hard to deduce the program from the output, but not impossible if we allow for human intuition and close, but not perfect deduction. Is `noise.png` low-complexity art? Is this a counterexample that contradicts Schmidhuber's thesis? How well did data compression do at measuring complexity?

  - [Lossy compression](https://en.wikipedia.org/wiki/Lossy_compression)
  - [Perceptual coding](https://en.wikipedia.org/wiki/Perceptual_coding)


- [Longest common subsequence](https://en.wikipedia.org/wiki/Longest_common_subsequence)

- [Run-length encoding](https://en.wikipedia.org/wiki/Run-length_encoding)

  Scan through the data replacing long contiguous runs of the same symbol with the symbol with the length of the run. 00000000110 becomes 0-8 1-2 0-1. 11 digits becomes 6. How might this do with the red circle and the noise? Different data compression techniques yield different compression ratios on data. In general, a good compression scheme "knows something" about the data it would compress.

- [Huffman encoding](https://en.wikipedia.org/wiki/Huffman_coding) (a simple, lossless data compression scheme)

  Compute the frequency of words in a corpus, creating a table of word-frequency pairs. Store this table along with a list of symbols in the original word order where more common words are represented by symbols with fewer bits. The 8 most common words might be stored as 000-111. "a", a common word, might be stored as 101 (5). If we can compress an object this way, it means that it is composed mostly of frequently-used words, perhaps punctuated by a few uncommon words. Something like this happens with language; We invent words (small) to represent common/shared complex ideas.

### Replica of Georg Nees' Schotter

<https://editor.p5js.org/kybr/sketches/m0JRd9zcf>

```txt
DATA   CONTEXT    BYTES
-----------------------------------------
R8     CODE:        780
RGB888 JPG:     1789719
RGB888 PNG:     2057058
RGB888 SCREEN: 44518080 (3160 * 4696 * 3)
```

```js
// 12 x 22

function setup() {
  createCanvas(1580, 2348);
  noLoop();
  rectMode(CENTER);
}

const N = 85;
const margin = 50;

function r() {
  return 2 * random() - 1;
}

function draw() {
  // https://redketchup.io/color-picker
  // https://collections.vam.ac.uk/item/O221321/schotter-print-nees-georg
  background("#DDD8D2");
  noFill();
  stroke(0);
  strokeWeight(4);
  let n = 0;

  for (let j = 0; j < 22; ++j) {
    for (let i = 0; i < 12; ++i) {
      let f = n / (12 * 22); // f: (0, 1)
      const x = i * N + r() * f * 3 + 290;
      const y = j * N + r() * f + 233;
      push();
      translate(x, y);
      rotate(r() * f);
      rect(0, 0, N, N);
      pop();
      n++;
    }
  }

  save("foo.png");
}

```
