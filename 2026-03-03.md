## Rules for sharing code

- **Always credit your collaborators.**

- _Attribution_: the action of regarding something as being caused by a person or thing.

- If you "borrow" code from somewhere (like we all do), then if falls on you to know who wrote it and attribute the code to them. If the code has a license, research that license and adhere to it. Honor copyright if the code is copyrighted. If you don't know enough about the code to pay attribution, then you don't deserve to use it. Investigate its provenance until you can attribute it correctly.

- If you share code with a fellow student, you must tell me and make it clear *in the code* that that is what you are doing. `// this part is from Frank Zappa; thanks, Frank!` and email me.

- If you use a LLM to generate code, then you must say so. Clearly mark the code as "LLM-generated", include the name and version of the LLM. Include the prompts in the code as comments.

Not following these rules is **plagiarism**.

## Fading away a drawing...

> How might I implement a graceful "fade" over time (so old brush strokes slowly vanish) without completely clearing the background every few frames, allowing the canvas to act as a temporary memory map of the audio movement rather than an ever-accumulating mess?

Well, there are basically two paths you can take. One might be better  than the other depending on your goal.

1. Use WEBGL, framebuffers, and  GLSL to draw and pull the color of pixels to some background color over  time---You may or may not use alpha blending. This is GPU-oriented system.
2. Store brush strokes in shapes or some data structure that "ages" by losing pieces of itself over time---This is more of a CPU-bound approach.

## Egg Drop Puzzle

You are standing at the bottom of a stairway that has 100 steps, with two identical eggs in your hand. **You must find the highest step S* from which a dropped egg will not break.**


Assume the following:

* If an egg doesn't break when dropped from a certain step, it will not break when dropped from any step below.

* If an egg breaks when dropped from a certain step, it will break when dropped from any step above.

* An egg may or may not break when dropped from the bottom step.

* An egg may or may not break when dropped from the top step.

* To reuse an egg you dropped that didn't break, you must descend the stairway to retrieve it.

* Descending the stairway does not require significant energy, but ascending it does. Hence, it is desirable to take as few upward steps as you can while determining S*.


a) Find a strategy that minimizes the total number of upward steps that, at worst, you must take to identify S*.

b) What is your strategy's worst-case total number of upward steps?
