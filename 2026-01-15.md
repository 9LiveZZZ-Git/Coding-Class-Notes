## Chaos: Making a New Science

Book by **James Gleick**

_Chaos: Making a New Science_ by James Gleick is a groundbreaking 1987 book that introduced the public to chaos theory, a field of science that studies complex systems with seemingly random behavior, revealing hidden patterns and order. Gleick explains concepts like the Butterfly Effect, fractals, and strange attractors through the stories of the scientists who developed them, making the subject accessible and fascinating. The book was a bestseller, a finalist for the Pulitzer Prize and National Book Award, and is considered a classic of popular science



## The Information: A History, a Theory, a Flood

Book by **James Gleick**

Gleick begins with the tale of colonial European explorers and their fascination with African talking drums and their observed use to send complex widely understood messages back and forth between villages, and over even longer distances by relay. Gleick transitions from the information implications of such drum signaling to the impact of the arrival of long-distance telegraph and then telephone communication to the commercial and social prospects of the Industrial Revolution west. Research to improve these technologies ultimately led to our understanding the essentially digital nature of information, quantized down to the unit of the bit (or qubit).

```js
function draw() {

```

```js
  //
```

**fragment shader**:

```glsl
precision highp float; // use float32

varying vec2 vPosition; // passed in from the vertex shader

uniform vec2 mouse; // passed in from javascript/p5

void main() {
  // sets the color of this pixel to (0, 0, 255) which
  // has the effect of making the _background_ blue
  //
  gl_FragColor = vec4(vec3(0.0, 0.0, 1.0), 1.0);

  // sets the color of this pixel to red (255, 0, 0) _if_
  // the pixel is very near (within +/- a) the x or y axes
  //
  const float a = 0.0025;
  if ((vPosition.x > -a && vPosition.x < a)
   || (vPosition.y > -a && vPosition.y < a)) {
    gl_FragColor = vec4(vec3(1.0, 0.0, 0.0), 1.0);
  }

  // sets the color of this pixel to yellow _if_ it is 
  // within 0.06 of the point (0.2, 0.5)
  //
  if (distance(vec2(0.2, 0.5), vPosition) < 0.06) {
    gl_FragColor = vec4(vec3(1.0, 1.0, 0.0), 1.0);
  }

  // sets the color of the pixels near the mouse
  //
  if (distance(mouse, vPosition) < 0.06) {
    gl_FragColor = vec4(vec3(1.0, 0.0, 1.0), 1.0);
  }
}
```



**vertex shader**:

```glsl
precision highp float; // use float32

attribute vec3 aPosition; // 3D "world" coordinate

varying vec2 vPosition; // 2D "clip space" coordinate

void main() {
  // maps the vertex to 3D "clip space"
  gl_Position = vec4(aPosition, 1.0);
  
  vPosition = gl_Position.xy; 
  // vPosition is passed on to the fragment shader
  // x is on (-1, 1) and y is on (-1, 1)
}
```



This form of drawing is *very* different than we might see in a p5 sketch. We are deciding the fate (the color) of each pixel given its position and maybe a bunch of other information. If we use this method to draw something impressively detailed or interesting, things quickly get challenging as the code can become inscrutable (see Ray Marching below).

But, this is the way to "make things fast". It is also the way to make lighting effects, realism, and detailed drawings. We face a tradeoff between _easy to understand but slow_ (p5) and _hard to understand but fast_ (GLSL). Also, there tends to be a bit of math when writing GLSL: linear algebra (translate, rotate, scale in 3D), trigonometry, and calculus (slash differential equations). - how does complex multiply work?

```glsl
vec2 complex_multiply(vec2 z0, vec2 z1) {
  return ???;
}
```

$$
z_0 \cdot z_1 \\
(x_0 + y_0i)(x_1 + y_1i) \\
x_0x_1 + y_0x_1i + x_0y_1i + y_0y_1 \\
x_0x_1 + y_0y_1 + (y_0x_1 + x_0y_1)i \\
\text{real part: } x_0x_1 + y_0y_1 \\
\text{imaginary part: } y_0x_1 + x_0y_1 \\
$$

```glsl
vec2 complex_multiply(vec2 z0, vec2 z1) {
  return vec2(
    z0.x * z1.x + z0.y * z1.y, // real part
    z0.y * z1.x + z0.x * z1.y // imaginary
  );
}
```

$$
\sum_{i=1}^{N} x \\
\text{float x = 1; float sum = 0; for (int i = 1; i < N; i++) { sum += x }}
