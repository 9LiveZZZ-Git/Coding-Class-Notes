## Coloring fractals

<https://mandelbrowser.y0.pl/tutorial/computing.html>

<https://chatgpt.com/share/696faee1-1288-800d-bb44-5a2029231487>

<https://editor.p5js.org/kybr/sketches/NR0obTTlL>

To draw the fractal, we must estimate if the pixel is in the set (it converges) or not (diverges). This gets us a two-color image. But, there is more information we can extract from the iteration process that we might use to decide the color of the pixel, making a more interesting image.

Each iteration jumps $z$ to a new position in the [complex plane](https://en.wikipedia.org/wiki/Complex_plane):

<https://editor.p5js.org/kybr/sketches/E9yDmPa4o>

We can see from the sketch above that there are statistics we can capture from the iterations. What about:

- the total length of the path?
- the average length of a segment in the path?
- the distance from starting point to ending point?
- how straight/curved is the path?

Also, we should use some alternative to the RGB color space. Perhaps [HSV](https://en.wikipedia.org/wiki/HSL_and_HSV) or [CIELAB](https://en.wikipedia.org/wiki/CIELAB_color_space).

$$
z_{n+1} = z_n - m \frac{f(z)}{f'(z)} \\
z_{n+1} = z_n - m \frac{z^3 - 1}{3z^2} \\
dz = m \frac{z^3 - 1}{3z^2} \\
z_{n+1} = z_n - dz
$$

```
vec2 dz =
   cx_div(
       cx_powr(z, 3.0) - cx_one(),
       cx_mulS(cx_sqr(z), 3.0)
   );
z = cx_sub(z, dz);
```
