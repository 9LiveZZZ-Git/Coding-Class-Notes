## Why Attribution Matters

When you write code for this course -- or for any creative project -- you are building on the work of others. Every library you use, every tutorial you follow, every snippet you borrow represents someone else's effort. **Attribution** is the practice of giving credit where credit is due.

Attribution is not just an academic rule. It is a professional norm, a legal requirement (in many cases), and a matter of basic respect for the creative community you are part of.

> *Attribution*: the action of regarding something as being caused by a person or thing.

---

## The Rules for This Course

From the syllabus:

1. **Always credit your collaborators.**

2. If you "borrow" code from somewhere (like we all do), then it falls on you to know who wrote it and attribute the code to them. If the code has a license, research that license and adhere to it. Honor copyright if the code is copyrighted. If you don't know enough about the code to pay attribution, then you don't deserve to use it. Investigate its provenance until you can attribute it correctly.

3. If you share code with a fellow student, you must tell the instructor and make it clear *in the code* that that is what you are doing.
   ```js
   // this part is from Frank Zappa; thanks, Frank!
   ```
   And email the instructor.

4. If you use a LLM to generate code, then you must say so. Clearly mark the code as "LLM-generated", include the name and version of the LLM. Include the prompts in the code as comments.

**Not following these rules is plagiarism.**

---

## What Is Plagiarism in Code?

Plagiarism in code means presenting someone else's work as your own. This includes:

- Copying code from a website without citing the source
- Submitting a classmate's code as your own
- Using LLM-generated code without disclosing it
- Paraphrasing code (changing variable names, rearranging lines) to disguise its origin
- Using code from a tutorial or textbook without acknowledging it

What is **not** plagiarism:

- Using code you wrote yourself
- Using code from a properly cited source
- Using a library or framework (these are meant to be used -- but you should understand your dependencies)
- Collaborating with a classmate **and clearly disclosing the collaboration**
- Using LLM-generated code **and clearly labeling it as such**

## How to Attribute Code

### Attributing Code from the Internet

When you use code from a website, tutorial, Stack Overflow answer, GitHub repository, or any other online source, include a comment in your code:

```js
// Based on Daniel Shiffman's "Coding Train" Perlin Noise tutorial
// Source: https://www.youtube.com/watch?v=8ZEMLCnn8v0
// Modified to use 3D noise instead of 2D

function generateTerrain() {
  // ... your adapted code ...
}
```

Include:
- **Who** wrote the original (name or username)
- **Where** you found it (URL)
- **What** you changed (if anything)

### Attributing Code from a Classmate

```js
// The following particle system was developed collaboratively with Jane Doe.
// Jane wrote the initial Particle class; I added the force accumulation and
// boundary wrapping. Collaboration disclosed to instructor via email on 2026-03-03.

class Particle {
  // ...
}
```

### Attributing Code from a Library

Libraries and frameworks are designed to be used, so you do not need to attribute every function call. However, if you use a lesser-known library or a specific technique from a library's documentation, mention it:

```js
// Using the math.js library for complex number arithmetic
// https://mathjs.org/
// Licensed under the Apache License 2.0
```

## LLM Usage Policy

### The Reality of LLMs in 2026

Large Language Models (ChatGPT, Claude, Gemini, Copilot, etc.) can generate code. Many of you are already using them. This is not inherently wrong -- but it must be transparent.

### This Course's Policy

If you use an LLM to generate any part of your code, you must:

1. **Mark the code clearly** as LLM-generated:
   ```js
   // --- BEGIN LLM-GENERATED CODE ---
   // Generated by: Claude 3.5 Sonnet (Anthropic), March 2026
   // Prompt: "Write a p5.js function that draws a spiral using polar coordinates"
   ```

2. **Name the LLM** and its version/model:
   ```js
   // LLM: ChatGPT (GPT-4), OpenAI
   // LLM: Claude 3.5 Sonnet, Anthropic
   // LLM: GitHub Copilot
   ```

3. **Include your prompts** as comments:
   ```js
   // Prompt 1: "How do I create a framebuffer in p5.js WEBGL mode?"
   // Prompt 2: "Now make it so the previous frame fades out gradually"
   // Prompt 3: "Add mouse interaction so I can draw into the framebuffer"
   ```

4. **Mark the end** of LLM-generated code:
   ```js
   // --- END LLM-GENERATED CODE ---
   ```

### Why This Matters

- **Learning**: If you do not understand the code, you are not learning. The point of this course is to develop *your* skills, not to demonstrate that an LLM can write code.
- **Academic integrity**: Passing off LLM-generated code as your own is plagiarism, just like copying from a human.
- **Professional practice**: In industry, undisclosed use of LLM-generated code can create legal liability (unclear copyright, potential license violations in training data).
- **Debugging**: If you do not understand code you submitted, you will not be able to fix it when it breaks or adapt it for your final project.

### The p5.js Community's Perspective

The p5.js project has published its own AI usage policy:

<https://github.com/processing/p5.js/blob/dev-2.0/AI_USAGE_POLICY.md>

And a thoughtful tutorial on the topic:

<https://beta.p5js.org/tutorials/criticalai1-chatting-with-about-code>

These resources explore the nuances of AI-assisted coding in a creative context. Reading them is strongly recommended.

### What Is Acceptable Use of LLMs in This Course

| Use Case | Acceptable? | Notes |
|---|---|---|
| Ask an LLM to explain a concept | Yes | This is like reading a textbook |
| Ask an LLM to debug your code | Yes, with attribution | Note what you asked and what it suggested |
| Ask an LLM to generate a complete sketch | Yes, with full attribution | You must label all generated code and include prompts |
| Submit LLM code without attribution | **No** | This is plagiarism |
| Use an LLM for your final project | Yes, with attribution | But you must deeply understand every line |
| Ask an LLM to write your project report | **No** | Written reports must be your own words |

## Open Source Licenses

### Why Licenses Matter

When someone publishes code, the default legal status is: **all rights reserved**. Without an explicit license, you technically cannot copy, modify, or distribute the code -- even if it is publicly visible on GitHub.

An **open source license** is a legal document attached to code that grants specific permissions. It tells you what you can and cannot do with the code.

### Common Open Source Licenses

#### MIT License

The most permissive common license. You can do almost anything with the code as long as you include the original copyright notice.

```
Permission is hereby granted, free of charge, to any person obtaining a copy
of this software... to deal in the Software without restriction, including
without limitation the rights to use, copy, modify, merge, publish, distribute,
sublicense, and/or sell copies...
```

**p5.js uses the LGPL license** (GNU Lesser General Public License), which is more restrictive than MIT but still allows you to use the library in your projects.

#### GPL (GNU General Public License)

If you use GPL-licensed code in your project, your entire project must also be released under the GPL. This is called "copyleft" -- it ensures that derived works remain open source.

#### Apache License 2.0

Similar to MIT but includes an explicit patent grant. Used by math.js and many large projects.

#### Creative Commons

Not for code, but for creative works (images, music, text, documentation). You will use a Creative Commons license for your final project's non-code materials.

### How to Check a License

1. Look for a `LICENSE` or `LICENSE.md` file in the project's root directory.
2. Check the project's README or website.
3. On npm packages, run `npm info <package> license`.
4. If there is no license, the code is **all rights reserved** -- do not use it without permission.

### How to Apply a License to Your Own Code

For your final project, you will choose a software license (for code) and a Creative Commons license (for creative works). Add a `LICENSE` file to your project repository:

```
MIT License

Copyright (c) 2026 Your Name

Permission is hereby granted, free of charge, to any person obtaining a copy
of this software and associated documentation files (the "Software"), to deal
in the Software without restriction, including without limitation the rights
to use, copy, modify, merge, publish, distribute, sublicense, and/or sell
copies of the Software, and to permit persons to whom the Software is
furnished to do so, subject to the following conditions:

The above copyright notice and this permission notice shall be included in all
copies or substantial portions of the Software.

THE SOFTWARE IS PROVIDED "AS IS", WITHOUT WARRANTY OF ANY KIND, EXPRESS OR
IMPLIED...
```

Choose your license at: <https://opensource.org/licenses>

Choose your Creative Commons license at: <https://creativecommons.org/share-your-work>

## A Practical Example: Full Attribution in a Sketch

Here is what a well-attributed p5.js sketch looks like:

```js
/**
 * Audio-Reactive Particle System
 * MAT 200C: Computing Arts -- Final Project
 * Author: Your Name
 * Date: March 2026
 * License: MIT
 *
 * Dependencies:
 *   - p5.js (LGPL) https://p5js.org
 *   - p5.sound (LGPL) https://p5js.org/reference/p5.sound/
 *
 * Credits:
 *   - Particle system structure based on Daniel Shiffman's
 *     "The Nature of Code" Chapter 4
 *     https://natureofcode.com/particles/
 *
 *   - FFT analysis approach adapted from p5.sound documentation
 *     https://p5js.org/reference/p5.FFT/
 *
 *   - Color palette inspired by a sketch by Jane Doe (classmate)
 *     Collaboration disclosed to instructor on 2026-02-15
 *
 * LLM Usage:
 *   - Claude 3.5 Sonnet (Anthropic) was used to generate the
 *     mapFrequencyToColor() function below (lines 87-102).
 *     Prompt: "Write a function that maps a frequency value (20-20000 Hz)
 *     to an HSB color, with low frequencies as warm colors and high
 *     frequencies as cool colors."
 */

// ... your code ...

// --- BEGIN LLM-GENERATED CODE ---
// Generated by: Claude 3.5 Sonnet (Anthropic), February 2026
// Prompt: "Write a function that maps a frequency value (20-20000 Hz)
//          to an HSB color, with low frequencies as warm colors and
//          high frequencies as cool colors."
function mapFrequencyToColor(freq) {
  let logFreq = log(freq / 20) / log(1000);
  let hue = map(logFreq, 0, 1, 0, 240);
  let saturation = 80;
  let brightness = 90;
  return color(hue, saturation, brightness);
}
// --- END LLM-GENERATED CODE ---

// ... your code ...
```

## Summary

1. **Always attribute your sources** -- code, ideas, collaborations.
2. **Label LLM-generated code** with the model name, version, and prompts.
3. **Understand the licenses** of code you use.
4. **Choose a license** for your own work.
5. **When in doubt, over-attribute.** No one has ever been penalized for giving too much credit.
