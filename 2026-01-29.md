## 2D rotations

```GLSL
vec2 scaleRotate(vec2 v, vec2 pivot, vec2 scale, float angle) {
    return
      pivot + (v - pivot)
        * scale
        * mat2(
            vec2(cos(angle), -sin(angle)),
            vec2(sin(angle),  cos(angle)));
}
```





















## Laplacian

The Laplacian (\(\Delta \) or \(\nabla ^{2}\)) is a differential operator that measures the difference between a function's value at a point and its average value over neighboring points, effectively calculating the local curvature or "flux density" of a scalar field. It acts as a multi-dimensional second derivative, detecting edges, calculating diffusion, and identifying local maxima/minima. 



**5-point:**

```GLSL
// 2D Laplacian of a texture (5-point stencil), per-channel.
// Returns: (Laplacian of RGBA) at uv.
vec4 laplacian(sampler2D tex, vec2 uv) {

    vec2 texel = 1.0 / vec2(textureSize(tex, 0));

    vec4 c  = texture(tex, uv);
    vec4 xp = texture(tex, uv + vec2(texel.x, 0.0));
    vec4 xm = texture(tex, uv - vec2(texel.x, 0.0));
    vec4 yp = texture(tex, uv + vec2(0.0, texel.y));
    vec4 ym = texture(tex, uv - vec2(0.0, texel.y));

    // Discrete Laplacian: sum(neighbors) - 4*center
    return (xp + xm + yp + ym) - 4.0 * c;
}
```



**3x3:**

```GLSL
// 2D Laplacian using a 3x3 (9-point) stencil.
// Weights give a more isotropic approximation than 5-point.
vec4 laplacian3(sampler2D tex, vec2 uv) {

    vec2 texel = 1.0 / vec2(textureSize(tex, 0));

    vec4 c  = texture(tex, uv);

    vec4 xp = texture(tex, uv + vec2( texel.x, 0.0));
    vec4 xm = texture(tex, uv + vec2(-texel.x, 0.0));
    vec4 yp = texture(tex, uv + vec2(0.0,  texel.y));
    vec4 ym = texture(tex, uv + vec2(0.0, -texel.y));

    vec4 xpyp = texture(tex, uv + vec2( texel.x,  texel.y));
    vec4 xpym = texture(tex, uv + vec2( texel.x, -texel.y));
    vec4 xmyp = texture(tex, uv + vec2(-texel.x,  texel.y));
    vec4 xmym = texture(tex, uv + vec2(-texel.x, -texel.y));

    // One common 9-point Laplacian stencil:
    //   [ 1  4  1 ]
    //   [ 4 -20 4 ]
    //   [ 1  4  1 ]
    //
    // (often normalized by /6.0 or / (texel^2) depending on use)
    return (xpyp + xpym + xmyp + xmym)       // diagonals * 1
         + 4.0 * (xp + xm + yp + ym)         // axis neighbors * 4
         - 20.0 * c;                         // center
}
```

## [Reaction-Diffusion Systems](https://en.wikipedia.org/wiki/Reaction%E2%80%93diffusion_system)

> ...mathematical models which correspond to several physical phenomena. The most common is the change in space and time of the concentration of one or more chemical substances: local chemical reactions in which the substances are transformed into each other, and diffusion which causes the substances to spread out over a surface in space. 
>
> ...solutions of reaction–diffusion equations display a wide range of behaviours, including the formation of travelling waves and wave-like phenomena as well as other self-organized patterns like stripes, hexagons or more intricate structure like dissipative solitons. Such patterns have been dubbed "[Turing patterns](https://en.wikipedia.org/wiki/Turing_pattern)".
